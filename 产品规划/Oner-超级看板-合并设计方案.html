1 合并矩阵：哪些方案和泳道天然互补

从12个保留方案中筛选与泳道看板有结构重叠的方案，分为「核心合并」「增强合并」「锦上添花」三层。

核心合并 创意2: 今日焦点

首页和看板共享「今日数据源」。泳道看板顶部增加"今日概览条"，展示今日到期+高优先+AI推荐——不再需要切到首页才知道今天该干什么。

→ 泳道顶部 TodayStrip 组件
核心合并 创意9: 卡片视觉权重

泳道内同一分类的卡片通过边框颜色、背景色、尺寸差异区分重要性。紧急=红色左边框+浅红底，高优先=橙色左边框，普通=默认样式，已完成=半透明。

→ 泳道卡片 4 种权重等级
核心合并 创意7: 乐观UI + 骨架屏

泳道拖拽（改状态+改分类）必须即时反馈。数据加载时显示骨架屏而非全屏spinner，泳道框架+空卡片占位先行出现。

→ useOptimistic hook + Skeleton 组件
增强合并 创意17: Toast 通知

替代所有 window.confirm()。拖拽到泳道新行后弹 Toast "已移到「学习」"；拖拽改状态后弹 "状态已更新"；操作失败弹 "网络异常，已回滚"。

→ useToast hook + Toast 组件
增强合并 创意18: 功能色彩分区

每个泳道行拥有专属色系——工作蓝、学习紫、生活绿、项目橙。行背景微妙着色，卡片边框取色于行，建立视觉一致性和空间感。

→ lane 级 CSS 变量 + 主题色注入
增强合并 创意1: Command Bar

看板的 Toolbar 从双行简化为单行 Command Bar。分类筛选移入泳道标签区（点击泳道标签筛选），状态筛选由列头天然承担，顶部只保留搜索+操作按钮。

→ CommandBar 替代 Toolbar
锦上添花 创意19: 移动端适配

移动端保持 SwipeStatusTabs 默认视图（列模式），横屏/平板切泳道模式。或通过「左滑切换分类泳道」手势模拟。

→ 响应式切换 + 手势引擎
锦上添花 看板原有优化（§10.2）

原报告中看板章节的建议也融入：卡片增加进度条/截止日期/子任务计数、列头计数动画、"今日筛选"开关、拖拽时目标格高亮+虚线边框。

→ 直接在泳道卡片中实现

3 乐观UI + 骨架屏 + Toast 效果

创意7和创意17在看板中的具体表现。加载时显示骨架，拖拽时即时更新，操作后Toast反馈。

🦴 骨架屏加载（替代全屏 Spinner）

看板数据加载时，先显示列框架+卡片占位。数据到达后 fade-in 填充。用户感知等待时间减少 50-60%。

5 建议实施顺序

按依赖关系和优先级排序，分 4 步落地。每步可独立上线验证。

1
基础设施（第1周）
先建 UX 基础设施，不依赖看板改动：

useToast hook + Toast 组件
Skeleton 骨架屏组件（Card/Board 变体）
useOptimistic 乐观更新 hook
ConfirmDialog 替代 window.confirm
全局微动画 CSS 变量

✓ 这一步对全产品都有益，不仅看板。
2
泳道核心组件（第2周）
新建 SwimlaneBoard 组件：

CSS Grid 布局：130px repeat(4, 1fr)
SwimLaneCell = Droppable 区域
复用 SortableNoteCard（加 visual weight 变体）
groupedByLane useMemo 计算
拖拽检测：同行=改状态，跨行=改分类
viewMode toggle 切换泳道/列视图

✓ 现有列视图完全不受影响。
3
今日概览条（第3周）
新建 TodayStrip 组件：

数据源：从 notes 中筛选今日到期 + 高优先 + 已过期
横向滚动卡片，点击跳转到泳道对应位置
与首页 TodayFocus 共享筛选逻辑
"今日到期" filter pill 联动

✓ 这一步同时增强首页和看板。
4
打磨 + 移动端（第4周）
视觉和交互细节打磨：

泳道行主题色注入（CSS 变量）
泳道折叠/展开动画
泳道排序（拖拽泳道标签）
移动端横屏检测 → 自动切泳道
骨架屏在看板加载中的应用
全局 SVG 图标替换（创意3）

✓ 锦上添花，可在后续迭代中渐进完成。
