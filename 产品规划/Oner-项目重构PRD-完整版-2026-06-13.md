## 1. 项目现状诊断

### 1.1 当前问题

| 问题类别 | 具体表现 | 严重程度 |
|---------|---------|---------|
| **架构耦合** | 所有功能（AI/密码/备忘）都在 `frontend/src/` 下，无模块边界 | 🔴 高 |
| **组件混乱** | 40+ 组件平铺在 `components/`，无分类、无层级 | 🟡 中 |
| **样式无系统** | Tailwind 随意使用，无 Design Token，深色/浅色靠手动切换 | 🔴 高 |
| **无设计规范** | 圆角/间距/阴影各处不一致，开发者各自发挥 | 🟡 中 |
| **无模块化入口** | `App.jsx` 静态 import 所有页面，无法按需加载 | 🔴 高 |

### 1.2 现有文件清单（需重构）

```
frontend/src/
├── App.jsx                    ← 需拆分为路由配置模块
├── main.jsx                   ← 需保留，增加模块注册
├── components/ (40+ files)    ← 需按模块重新组织
├── pages/ (12 files)          ← 需按业务模块拆分
├── hooks/ (8 files)           ← 需按模块归类
├── utils/ (4 files)           ← 需提升为共享工具层
└── styles/ (1 file)           ← 需扩展为完整 Design System
```

## 2. 重构目标与原则

### 2.1 核心目标

1. **模块化**：每个业务功能独立成模块，可独立开发、测试、上线
2. **设计规范统一**：建立 Design System，所有组件遵循统一规范
3. **性能优化**：首屏只加载核心模块，其他模块懒加载
4. **可维护性**：目录结构即文档，新人一看就懂

### 2.2 设计原则

- **移动端优先**：所有设计优先考虑 375px 宽度，再适配大屏
- **零学习成本**：用户打开就能用，不弹教程
- **快即是好**：核心路径（新建备忘）≤ 2 步，响应 ≤ 200ms
- **离线优先**：本地 SQLite 优先，网络只是同步手段

## 5. 各模块 UI 设计规范

### 5.1 全局设计约束

**所有模块必须遵守以下规范，否则 PR 不通过：**

| 约束项 | 规范 | 检查方式 |
|--------|------|---------|
| 圆角 | 统一使用 `var(--radius)` / `var(--radius-sm)` / `var(--radius-lg)` | ESLint 规则 |
| 间距 | 统一使用 `var(--space-*)` Token | 代码 Review |
| 颜色 | 只允许使用 Design Token，禁止硬编码色值 | CI Lint |
| 字体 | 标题 `var(--font-title)`，正文 `var(--font-body)` | 视觉验收 |
| 响应式 | 移动端 375px 必须正常，平板 768px 适配，桌面 1024px+ 优化 | 多设备截图 |

### 6.2 基础组件规范

#### Button 按钮

```jsx
// 使用方式
<Button variant="primary" size="md" onClick={...}>保存</Button>
<Button variant="ghost" size="sm" icon={<Icon/>}>取消</Button>

// 规格
variant: "primary" | "secondary" | "ghost" | "danger" | "success"
size: "sm" | "md" | "lg"
disabled: boolean
loading: boolean
icon: ReactNode (可选，放在文字前)
iconOnly: boolean (仅图标按钮，正方形)
```

| Variant | 背景 | 文字 | Border | Hover | Active |
|---------|------|------|--------|-------|--------|
| primary | `var(--color-primary-500)` | white | none | `var(--color-primary-600)` + 上移-1px | `var(--color-primary-700)` |
| secondary | transparent | `var(--text-secondary)` | `1px solid var(--border-strong)` | `var(--bg-surface-2)` | `var(--bg-page)` |
| ghost | transparent | `var(--text-secondary)` | none | `var(--bg-surface-2)` | `var(--bg-page)` |
| danger | `var(--color-error)` | white | none | `oklch(from var(--color-error) l-0.05 c h)` | `oklch(from var(--color-error) l-0.1 c h)` |
| success | `var(--color-success)` | white | none | `oklch(from var(--color-success) l-0.05 c h)` | `oklch(from var(--color-success) l-0.1 c h)` |

尺寸规格：
- sm: 高度 28px，padding `6px 12px`，字号 `var(--font-sm)`
- md: 高度 36px，padding `8px 16px`，字号 `var(--font-base)`
- lg: 高度 44px，padding `10px 20px`，字号 `var(--font-lg)`

#### Input 输入框

```jsx
<Input 
  label="标题" 
  placeholder="输入标题..." 
  value={title} 
  onChange={setTitle}
  error="标题不能为空"
  helper="最多 100 个字符"
/>
```

规格：
- 高度（md）：36px
- 边框：`1px solid var(--border-default)`
- 圆角：`var(--radius-sm)`
- Focus：边框 `var(--border-accent)` + `box-shadow 0 0 0 3px var(--color-primary-100)`
- Error：边框 `var(--color-error)` + 文字 `var(--color-error)`
- Disabled：背景 `var(--bg-surface-2)`，光标 `not-allowed`，opacity 0.6

#### Card 卡片

```jsx
<Card padding="md" hoverable onClick={...}>
  <CardHeader>Title</CardHeader>
  <CardBody>Content</CardBody>
  <CardFooter>Actions</CardFooter>
</Card>
```

规格：
- 背景：`var(--bg-surface)`
- 边框：`1px solid var(--border-default)`
- 圆角：`var(--radius)`
- 阴影：`var(--shadow-sm)`
- Hover（hoverable=true）：阴影 `var(--shadow-md)`，上移 -2px
- Padding（md）：`var(--space-4)` (16px)

#### Modal 对话框

```jsx
<Modal open={isOpen} onClose={close} title="确认删除" size="md">
  <p>确定要删除这条备忘吗？</p>
  <ModalFooter>
    <Button variant="ghost" onClick={close}>取消</Button>
    <Button variant="danger" onClick={confirmDelete}>删除</Button>
  </ModalFooter>
</Modal>
```

规格：
- 遮罩：背景 `rgba(0,0,0,0.5)`，backdrop-blur 4px
- 容器：背景 `var(--bg-surface)`，圆角 `var(--radius-lg)`，最大宽度（sm=400px, md=560px, lg=720px）
- 动画：fadeIn 150ms + slideUp 200ms
- 关闭：点击遮罩 / ESC键 / X按钮

#### Badge 标签

```jsx
<Badge variant="primary" size="sm">新</Badge>
<Badge variant="success" dot>已完成</Badge>
```

规格：
- 高度（sm）：18px，padding `0 6px`，字号 11px
- 高度（md）：22px，padding `0 8px`，字号 12px
- variant 颜色同 Button variant
- dot 模式：左侧 6px 圆点，无文字

#### Toast 提示

```jsx
<Toast type="success" message="保存成功" onClose={close} />
```

规格：
- 位置：顶部居中，距顶 `var(--space-4)`
- 宽度：320px
- 自动关闭：success 3s，error 5s，info 4s
- 动画：slideDown 200ms 进入，fadeOut 150ms 离开

### A.3 常见问题 FAQ

**Q: 我可以用 Tailwind 吗？**
A: 可以，但颜色/间距/圆角必须用 Token 对应的 Tailwind 类，不能用任意值。例如 `bg-primary-500` ✅，`bg-[#ff0000]` ❌。

**Q: 深色主题怎么测试？**
A: 在浏览器 DevTools 中手动给 `<html>` 加 `data-theme="dark"` 属性，或用 Oner 设置页切换。

**Q: 移动端测试必须用真机吗？**
A: Chrome DevTools 设备模拟器可以，但最终验收建议真机（iPhone/Android 各一台）。
