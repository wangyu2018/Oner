设计总览 — 三个子页面的信息架构
当前 AI 洞察面板（AIInsightPanel.jsx）仅有三张简略卡片，点击后触发 onInsightAction 回调但无独立页面。本方案为三个子功能设计完整独立页面，路由分别为 /ai/overdue、/ai/weekly、/ai/associations。
📅

路由：/ai/overdue
核心：时间线 + 紧急度分组
操作：一键延期 / 标记完成
📊
本周概览
路由：/ai/weekly
核心：统计卡片 + 趋势图 + 热力图
操作：查看趋势 / 导出报告
🔗
智能关联
路由：/ai/associations
核心：关联图谱 + 列表 + AI解释
操作：确认关联 / 整理建议
🏗 页面导航架构

设计原则：三个子页面共享同一套 AI 洞察设计语言——紫/靛渐变主题色、圆角卡片、毛玻璃效果。页面间通过顶部 Tab 栏快速切换，无需返回上级页面。移动端通过底部导航「AI」tab 进入洞察首页，再切换子页面。

1
📅 到期提醒页面 — /ai/overdue
核心目标：让用户一眼看清哪些事项已过期/即将到期，并能快速处理。设计要点：按紧急度分组（已过期/今天/明天/本周），时间线视图，每条事项附带快速操作按钮无需进入详情页即可处理。

前端实现路径 — 对应 Oner 现有代码结构
基于现有路由（App.jsx）和组件体系，给出最小改动实现方案。每个页面新增一个 Page 组件 + App.jsx 加一条 Route + AIInsightPanel 回调改为路由跳转。
App.jsx — 新增 3 条路由
import OverduePage from './pages/OverduePage';
import WeeklyPage from './pages/WeeklyPage';
import AssociationsPage from './pages/AssociationsPage';
// 在 <Routes> 中新增：
<Route path="/ai/overdue" element={
 <AuthGuard><OverduePage/></AuthGuard>
/>
<Route path="/ai/weekly" element={
 <AuthGuard><WeeklyPage/></AuthGuard>
/>
<Route path="/ai/associations" element={
 <AuthGuard><AssociationsPage/></AuthGuard>
/>
Home.jsx — onInsightAction 回调升级
// 修改 AIInsightPanel 的回调：
const handleInsightAction = (action) => {
 switch(action) {
  case 'view-overdue':
   navigate('/ai/overdue');
   break;
  case 'view-statistics':
   navigate('/ai/weekly');
   break;
  case 'organize-tasks':
   navigate('/ai/associations');
   break;
 }
}

实现总结：三个子页面各需新增 1 个 Page 组件 + 2-3 个子组件，总计约 10 个新文件。App.jsx 增加 3 条 Route（约 30 行），Home.jsx 的 handleInsightAction 改为 navigate() 跳转（约 10 行改动）。改动量极小、风险可控，且三页面可独立开发和测试，不影响现有功能。
