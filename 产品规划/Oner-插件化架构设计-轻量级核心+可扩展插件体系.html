① 架构总览 — 三层设计
核心原则：Kernel（内核）只提供最基础能力，所有业务功能通过 Plugin（插件） 接入。
插件可以独立开发、测试、发布、启用/禁用，不影响核心稳定性。

💡 核心设计理念

Kernel 不依赖任何 Plugin，Plugin 只依赖 Kernel 提供的 API。
用户安装 Oner 后，默认只有 Notes Plugin（笔记核心）。
AI、Memo、Password 等功能都是可选插件，用户可以选择性安装。

② Plugin 生命周期 — 4 个阶段
每个插件都遵循相同的生命周期，Kernel 统一管理
1

install() — 安装阶段

插件第一次被添加到项目时调用。执行数据库迁移（创建表）、初始化配置、注册权限声明。
只执行一次，数据持久化到插件元数据库。

2

activate() — 激活阶段

每次应用启动时调用（如果插件处于 enabled 状态）。注册前端路由、后端 API 路由、侧边栏菜单项、命令面板命令。
这是插件"生效"的阶段。

3

deactivate() — 停用阶段

用户禁用插件时调用。注销路由、菜单项、命令，停止后台服务。数据保留，可重新激活。
插件功能消失，但用户数据不丢失。

4

uninstall() — 卸载阶段（可选）

用户彻底删除插件时调用。可选择是否删除数据表和数据。如果保留数据，重新安装时可恢复。
这是唯一可能丢失数据的操作，需用户确认。

③ Plugin Manifest — plugin.json 规范
每个插件根目录必须有 plugin.json，Kernel 根据它了解插件的能力和依赖

④ Kernel API — Plugin 能调用的核心接口
Kernel 暴露一组稳定的 API，插件通过这套 API 注册功能、存取数据、响应事件
// Kernel API TypeScript 类型定义（插件作者参考）
interface OnerKernel {
  // === 前端 API ===
  frontend: {
    // 注册路由（插件加载时调用）
    registerRoute(route: PluginRoute): void;

    // 注册侧边栏菜单项
    registerSidebarItem(item: SidebarItem): void;

    // 注册命令面板命令
    registerCommand(cmd: CommandItem): void;

    // 注册设置页面 Tab
    registerSettingsTab(tab: SettingsTab): void;

    // 注册底部导航项（移动端）
    registerBottomNavItem(item: BottomNavItem): void;

    // 渲染 Slot（在指定位置注入组件）
    registerSlot(slotName: string, component: React.Component): void;
  };

  // === 后端 API ===
  backend: {
    // 注册 Express 路由
    registerRoute(path: string, router: Express.Router): void;

    // 注册数据库迁移
    registerMigration(migration: Migration): void;

    // 注册定时任务
    registerCron(job: CronJob): void;
  };

  // === 数据 API ===
  data: {
    // 调用其他插件的 API（通过插件 ID）
    callPluginAPI(pluginId: string, method: string, params: any): Promise<any>;

    // 读取插件配置
    getConfig(key: string): any;
    setConfig(key: string, value: any): void;
  };

  // === 事件 API ===
  events: {
    // 监听事件
    on(event: string, handler: EventHandler): void;
    // 触发事件（广播给其他插件）
    emit(event: string, data: any): void;
  };
}

⑥ 插件实现示例 — AI 插件前端入口
展示一个插件如何通过 Kernel API 注册路由、侧边栏、命令
// plugins/ai/frontend/index.jsx — AI 插件前端入口
import React from 'react';
import AIChatPage from './pages/AIChatPage.jsx';
import OverduePage from './pages/OverduePage.jsx';
import WeeklyPage from './pages/WeeklyPage.jsx';
import AssociationsPage from './pages/AssociationsPage.jsx';

// 插件激活时调用 — Kernel 传入 kernelAPI
export function activate(kernelAPI) {
  const { frontend, events } = kernelAPI;

  // 1. 注册前端路由
  frontend.registerRoute({
    path: '/ai',
    component: AIChatPage,
    exact: false,  // 支持 /ai/* 子路由
  });

  frontend.registerRoute({
    path: '/ai/overdue',
    component: OverduePage,
  });

  frontend.registerRoute({
    path: '/ai/weekly',
    component: WeeklyPage,
  });

  frontend.registerRoute({
    path: '/ai/associations',
    component: AssociationsPage,
  });

  // 2. 注册侧边栏菜单项
  frontend.registerSidebarItem({
    id: 'ai-chat',
    label: 'AI 助手',
    icon: '🤖',
    path: '/ai',
    weight: 90,  // 控制排序，数字越大越靠下
    badge: null,  // 可选的 badge 计数
  });

  // 3. 注册命令面板命令
  frontend.registerCommand({
    id: 'ai:summary',
    label: 'AI 总结当前笔记',
    icon: '🤖',
    action: () => {
      events.emit('ai:request-summary', { noteId: null });
    },
    shortcut: 'Cmd+Shift+A',
  });

  // 4. 注册底部导航（移动端）
  frontend.registerBottomNavItem({
    id: 'ai',
    label: 'AI',
    icon: Bot,  // Lucide icon name
    path: '/ai',
    weight: 80,
  });

  // 5. 监听事件（笔记保存时触发 AI 分析）
  events.on('note:saved', async (data) => {
    // 调用后端 AI 插件进行分析
    await kernelAPI.data.callPluginAPI(
      'oner.plugin.ai.backend',
      'analyzeNote',
      { noteId: data.noteId }
    );
  });

  // 返回清理函数（deactivate 时调用）
  return function cleanup() {
    console.log('AI Plugin frontend deactivated');
  };
}

// 插件停用时调用
export function deactivate(kernelAPI) {
  // Kernel 会自动注销该插件注册的所有路由/菜单
  // 这里只做插件自身的清理工作（定时器、WebSocket 等）
  kernelAPI.backend.stopCron('ai-daily-insight');
}
⑦ 后端插件示例 — AI 插件后端入口
后端插件注册 Express 路由、数据库迁移、定时任务
// plugins/ai/backend/index.js — AI 插件后端入口
import express from 'express';
import aiRoutes from './routes/ai.js';
import migration001 from './migrations/001_create_ai_insights.js';

export function activate(kernelAPI) {
  const { backend, data, events } = kernelAPI;

  // 1. 注册后端 API 路由（自动挂载到 /api/plugin/ai/...）
  const router = express.Router();
  router.use('/ai', aiRoutes);
  backend.registerRoute('/api/plugin/ai', router);

  // 2. 注册数据库迁移
  backend.registerMigration({
    version: 1,
    name: 'create_ai_insights_table',
    up: migration001.up,
    down: migration001.down,
  });

  // 3. 注册定时任务（每天 8:00 生成 AI 洞察）
  backend.registerCron({
    id: 'ai-daily-insight',
    schedule: '0 8 * * *',  // cron 表达式
    handler: async () => {
      const users = await data.db.all('SELECT id FROM users');
      for (const user of users) {
        await kernelAPI.data.callPluginAPI(
          'oner.plugin.ai.backend',
          'generateDailyInsight',
          { userId: user.id }
        );
      }
    },
  });

  // 4. 注册可被其他插件调用的 API 方法
  data.registerAPIMethod('analyzeNote', async (params) => {
    // ... 实现笔记分析逻辑
    return { insights: [] };
  });

  // 5. 监听事件
  events.on('note:created', async (data) => {
    // 新笔记创建后，触发 AI 关联分析
  });

  console.log('✅ AI Plugin backend activated');
}

export function deactivate(kernelAPI) {
  // Kernel 自动注销路由和定时任务
  console.log('AI Plugin backend deactivated');
}
⑧ Kernel 核心 — PluginManager 简化实现
Kernel 的 PluginManager 负责发现、加载、激活、停用插件
// kernel/frontend/PluginManager.js — 前端插件管理器（简化版）
import React from 'react';
import kernelAPI from './kernel-api.js';

class PluginManager {
  plugins = new Map();  // pluginId → pluginState
  routes = new Map();  // routePath → component
  sidebarItems = new Map();  // itemId → itemConfig
  commands = new Map();  // cmdId → cmdConfig

  // 扫描 plugins/ 目录，发现所有插件
  async discoverPlugins() {
    const pluginDirs = await window.electronAPI?.listPlugins()
      ?? [];  // 后端提供的 API

    for (const dir of pluginDirs) {
      try {
        const manifest = await import(`/plugins/${dir}/plugin.json`);
        this.plugins.set(manifest.id, {
          manifest,
          status: 'discovered',  // discovered → installed → active
          frontendModule: null,
        });
      } catch (e) {
        console.warn(`Failed to load plugin manifest: ${dir}`, e);
      }
    }
  }

  // 激活一个插件（调用其 activate 函数）
  async activatePlugin(pluginId) {
    const plugin = this.plugins.get(pluginId);
    if (!plugin || plugin.status === 'active') return;

    // 动态 import 插件前端入口
    const module = await import(`/plugins/${pluginId}/frontend/index.jsx`);
    plugin.frontendModule = module;

    // 调用插件的 activate 函数，传入 kernelAPI
    const cleanup = await module.activate(kernelAPI);
    plugin.cleanup = cleanup;
    plugin.status = 'active';

    // 触发重新渲染（通知 App 更新路由/菜单）
    this.emitChange();
  }

  // 停用插件
  async deactivatePlugin(pluginId) {
    const plugin = this.plugins.get(pluginId);
    if (!plugin || plugin.status !== 'active') return;

    // 调用插件 deactivate
    await plugin.frontendModule?.deactivate?.(kernelAPI);

    // 调用 cleanup（activate 返回的清理函数）
    await plugin.cleanup?.();

    // 注销该插件注册的所有内容
    this.unregisterAllByPlugin(pluginId);

    plugin.status = 'installed';
    this.emitChange();
  }

  // 供 Kernel API 调用：注册路由
  registerRoute(pluginId, route) {
    this.routes.set(route.path, { ...route, pluginId });
  }

  // 供 App.jsx 使用：获取所有已注册路由（动态生成 <Route>）
  getRoutes() {
    return [...this.routes.values()];
  }

  // 供 Sidebar.jsx 使用：获取所有侧边栏项（已排序）
  getSidebarItems() {
    return [...this.sidebarItems.values()]
      .sort((a,b) => a.weight - b.weight);
  }
}

export default new PluginManager();
