问题背景与设计目标
当前 Oner 备忘模块只有「列表堆积」，缺少知识之间的连接层。随着备忘数量增长，用户陷入"记了找不到、找到不关联"的困境。
😞 痛点一：堆积难查

备忘一多就变成「信息黑洞」——全是列表，无分类入口，只能靠搜索，而很多时候连关键词都想不起来。

🔗 痛点二：知识孤岛

相关备忘互不知晓，同一个话题的想法分散在不同时间，无法看到「想法是如何演化的」，知识无法沉淀。

🧠 痛点三：无思维脉络

缺少「思路发展路径」的可视化。用户记录了调研→分析→结论三步，但三者之间的关联只存在于用户脑子里。

共享标签 权重 25%

同一时间段创建、以及用户曾在同一会话中打开过的备忘，关联度加权。

思维链视图 — 看见想法如何演化
备忘不只是「时间点」，而是「演化路径上的节点」。思维链将相互关联的备忘按「想法演化关系」组织为可视化树状结构，帮助用户看清一个话题从萌芽到行动的完整脉络。
AI 自动建链 可手动调整 导出大纲
🔑 节点类型说明
起点 最初的想法/问题/灵感，链的根节点
延伸 对上级节点的进一步探索/调研
细化 对某个方面的深度分析/拆解
结论 汇总多条探索后得出的判断
行动 从思考到执行的落地计划
🤖 AI 如何识别关系
引用信号：备忘中出现「基于...」「参考...」「延伸自...」等关键词，或直接链接了另一条备忘 ID。
时序信号：同一话题的备忘按创建时间自动排列，时间早的倾向「起点/延伸」，时间晚的倾向「结论/行动」。
内容信号：备忘正文开头的句式——「问题：」→起点，「发现：」→延伸，「因此：」→结论，「下一步：」→行动。
用户可修正：拖拽节点调整父子关系，或点击节点类型标签手动修改，AI 会从修正中学习。

💡 设计决策：思维链不是强制填写的模板字段，而是 AI 自动从已有备忘中「发现并构建」的关系图谱。用户零配置即可使用，AI 先生成初稿，用户再按需调整。「AI 补全链条」功能会扫描所有备忘，将孤立的备忘找到合适的链条接入，或自动创建新的链条起点。节点可折叠/展开，支持拖拽重排父子关系，「导出为大纲」直接生成 Markdown 层级结构。

交互规格说明
开发实现参考规格，含触发条件、API 调用时机、性能指标与降级策略。
🔗 智能关联 — 规格
触发条件 进入详情后 1.5s / 编辑停顿 1.5s（内容变化才触发）
API 调用 POST /api/memo/:id/related
Body: { limit: 5 }
推荐数量 默认展示 3 条，「查看全部」展开最多 10 条
相似度阈值 ≥ 50% 才展示，避免低质量推荐
加载状态 骨架屏 3 格占位，不使用全屏 Spinner
降级策略 AI 不可用时，仅按标签精确匹配，显示「基于标签匹配」说明
手动关联 存入 memo_links 表，手动关联优先级高于 AI 推荐
性能指标 API 响应 ≤ 800ms (P90)，骨架屏消失在 800ms 内
🧠 思维链 — 规格
数据结构 memo_chain_nodes { id, memo_id, parent_id, type, ai_confidence }
AI 建链时机 手动触发（「AI 补全链条」按钮）+ 每天凌晨全量重算
节点类型 起点 / 延伸 / 细化 / 结论 / 行动（AI 标注，可手动修改）
层级深度 最多 5 层，超出自动提示「考虑合并层级」
入口 备忘列表右上角「链」图标 + 备忘详情关联面板「在思维链中查看」
操作 拖拽调整父子 / 点击修改类型 / 添加下一步节点 / 导出 Markdown 大纲
角标联动 列表中「🧠 在链中」角标点击直接跳转至该备忘在链中的位置
降级策略 AI 不可用时，用户仍可手动构建链条
📂 分类快速浏览 — 规格
视图切换 时间线（默认）· 分类磁贴 · 思维链，存入 localStorage
分类筛选 点击左侧分类名，右侧内容区实时过滤，不刷新页面
搜索范围 仅搜索当前分类内容（标题 + 正文 + 标签），实时模糊匹配
时间分组 今天 / 昨天 / 本周 / 上周 / 更早 · 空时不渲染组头
快速筛选 本周 / 有关联 / 有任务 / 思维链 四个胶囊 Tab
关联角标 「🔗 N条」角标直接点击展开关联面板（不跳转详情）
空分类 未分类数量为 0 时不渲染该分类（或弱化显示）
性能 分类切换渲染 ≤ 100ms（虚拟列表 when > 100 条）
