# 产品脑暴记录 — Oner V1.0 差异化方向探索

**主题**：Oner 如何围绕"桌面优先 × 极简交互 × 多端同步"打造差异化个人备忘/任务管理体验
**HMW**：我们如何让追求效率的个人用户，在桌面端以最短路径完成"想到→记录→管理→回顾"的完整闭环，同时不牺牲跨设备一致性？
**日期**：2026-04-29
**产出创意数**：18个（筛选后保留15个）
**参与框架**：SCAMPER、5Why根因追问、竞品启发法、约束创新法、反向脑暴法

---

## 一、问题聚焦

### 追问清单确认

| 维度 | 确认结论 |
|------|---------|
| **谁有这个问题？** | 个人效率用户，以轻度记录者为主（学生/职场白领），追求"想到即记"零摩擦体验 |
| **现在怎么解决的？** | 用系统备忘录、微信文件传输助手、或复杂工具（Notion/Todoist）——前者太简陋，后者太重 |
| **为什么重要？** | 个人备忘是高频低粘场景，3秒创建门槛决定了用户是否愿意打开；多端不一致会导致"记了找不到" |
| **什么算成功？** | 用户每天打开≥3次，创建路径≤3秒，跨设备数据零感知同步，零学习成本 |
| **约束条件** | 1-3人小团队；Electron/React/Node.js/SQLite 技术栈已定；定位轻量，不做复杂协作 |

### 产品现状速览（基于代码库扫描）

- **已就绪**：Markdown编辑器、标签系统、5状态流转、4级优先级、截止日、自动保存、撤销删除、JWT认证、设备会话管理、Electron托盘/全局快捷键/通知/自启、30秒轮询同步、数据导出
- **明显缺口**：无搜索功能、无离线支持、无主动提醒、模态弹窗打断流、必须登录才能使用、同步延迟30秒

## 三、Top 3 推荐创意

### 创意1：智能提醒引擎（Quick Win — 立即见效）

**描述**：基于现有截止日期和优先级体系，在Electron主进程中增加定时检查逻辑。当任务截止前到达配置的提前量时，通过系统Notification API推送桌面通知。默认只提醒"高"和"紧急"优先级的任务，用户可在设置中调整提前量和免打扰时段。

**解决的痛点**：用户给任务设置了截止日期，但由于没有主动提醒，最终还是遗忘。这是当前Oner从"记录工具"迈向"任务管理工具"的关键缺口。

**预期效果**：高/紧急优先级任务的按时完成率预计提升30%以上；桌面通知的到达率远高于应用内红点。

**实现路径**：
1. 后端：在notes表增加`reminder_sent_at`字段，避免重复提醒
2. Electron主进程：每5分钟查询一次本地或服务器数据，筛选即将到期的任务
3. 通知触发：调用`new Notification({ title, body })`，点击通知可直接打开对应笔记
4. 设置面板：新增"提醒设置"页，支持提前量选择和免打扰时段

**验证方式**：
- 第1周：内部测试，确保通知在Win/Mac稳定触发
- 第2-3周：灰度发布给50%活跃用户，对比实验组与对照组的任务完成率
- 成功指标：实验组高优先级任务完成率提升≥20%

**风险与应对**：
- 风险：提醒过于频繁导致用户关闭系统通知权限
- 应对：默认仅提醒高/紧急任务；首次提醒时附带"调整提醒设置"按钮；提供"今日不再提醒"快捷操作

---

### 创意2：全文搜索 + 智能排序（Quick Win — 补全核心能力）

**描述**：在现有笔记列表上方增加搜索框，支持标题和内容全文检索。后端利用SQLite FTS5扩展实现高效搜索，前端结果按"编辑时间近→优先级高→匹配度强"综合排序。搜索支持`#标签名`和`status:todo`等过滤语法。

**解决的痛点**：目前Oner没有任何搜索能力。当笔记超过20条时，用户只能通过标签和状态筛选来查找，路径长且不够精准。这是从"可用"到"好用"的基础能力缺口。

**预期效果**：笔记查找路径从"滚动浏览/多步筛选"缩短到"输入关键词→回车"；用户平均查找时间从15秒降至3秒。

**实现路径**：
1. 后端：在SQLite中创建FTS5虚拟表，与notes表同步
2. API：新增`GET /api/notes/search?q=关键词&tag=工作&status=todo`
3. 前端：Toolbar增加搜索框，输入时实时请求（300ms防抖），结果直接替换CardWall内容
4. 排序算法：综合得分 = 匹配度×0.4 + 优先级权重×0.3 + 时间衰减×0.3

**验证方式**：
- 技术Spike：1天内验证sql.js（Node.js SQLite）是否支持FTS5模块加载
- 如果sql.js不支持FTS5，退化为LIKE模糊搜索+前端排序（降级方案仍可用）
- 上线后埋点：统计搜索功能的使用频率和平均结果点击率
- 成功指标：50%以上的活跃用户每周至少使用1次搜索

**风险与应对**：
- 风险：sql.js的WASM版本可能不支持FTS5扩展
- 应对：已准备降级方案（LIKE + 前端排序）；如使用Node.js原生sqlite3则无此问题

### 创意3：免登录本地优先模式（Big Bet — 长期差异化）

**描述**：重构数据层，使Oner在桌面端默认以本地SQLite运行，用户首次打开应用无需注册即可创建笔记。当用户产生多端同步需求时，通过设置页一键注册/登录，本地数据可选择性上传至服务器。已登录用户也可以随时切换到"仅本地模式"。

**解决的痛点**：
1. 注册门槛是最大激活流失点——很多用户只是想"试试看"，看到注册表单直接关闭
2. "隐私可控"是核心价值主张之一，但当前server-first架构与此矛盾
3. 与Obsidian、Logseq等本地优先工具相比，Oner在隐私维度缺乏竞争力

**预期效果**：新用户激活率（打开应用→创建第一条笔记）预计从当前的"未知/假设较低"提升至70%以上；产品定位从"又一个云端笔记"转变为"你的数据你做主"。

**实现路径**：
1. 抽象存储层：定义统一接口`StorageProvider`，实现`LocalStorageProvider`和`RemoteStorageProvider`
2. Electron端：默认使用本地SQLite（通过better-sqlite3或现有sql.js），完全脱离网络可用
3. 同步层改造：同步作为可选模块，检测到网络+用户已登录时，执行双向增量同步
4. 冲突策略：时间戳优先（last-write-wins）+ 手动合并UI（初期简单处理）
5. 数据迁移：本地模式注册时，提供"上传本地数据"选项；退登时提供"保留本地数据"选项

**验证方式**：
- 第1阶段（2周）：技术方案评审 + 数据层原型验证
- 第2阶段（4周）：开发最小可用版本（本地模式+可选同步）
- 第3阶段（2周）：找10个目标用户做可用性测试，观察从下载到创建第一条笔记的漏斗
- 成功指标：本地模式用户的次日留存≥50%；从打开到创建首条笔记的转化率≥70%

**风险与应对**：
- 风险：架构改动大，可能引入数据不一致bug，影响现有已注册用户
- 应对：渐进式迁移——现有登录态用户保持server-first行为不变；新安装用户默认local-first；提供显式切换开关；迁移期间每日自动备份
- 风险：本地数据丢失（硬盘损坏/重装系统）
- 应对：本地模式下定期提示用户导出备份；注册时强烈建议开启云同步作为备份

## 四、被放弃的创意（存档参考）

| 创意 | 放弃原因 |
|------|---------|
| **智能自动分类（AI）** | 需要NLP模型或外部API，与"极简"和"小团队"约束冲突；且用户不一定信任AI分类结果 |
| **笔记双向链接（[[笔记标题]]）** | 属于知识管理重度功能，偏离"轻量备忘"产品定位；会显著增加交互复杂度 |
| **WebSocket实时推送** | 虽然体验好，但当前轮询在单人低并发场景下足够；可延后到用户量增长后再投入 |
| **桌面悬浮窗Top3** | 创意价值高，但实现复杂（Electron子窗口管理、跨平台适配、与主窗口状态同步）；列为V1.1候选 |
| **习惯打卡视图** | 可通过"每日重复任务+状态流转"间接实现，非当前阶段的独立功能 |
| **轻量项目/区域层级** | 标签系统目前够用，增加层级会带来架构和交互复杂度；等标签数量>20个/用户时再考虑 |

## 五、发散过程记录（供团队回顾）

### SCAMPER 七维度关键产出

- **S-替换**：用角落快速面板替换模态编辑器；用WebSocket替换轮询
- **C-组合**：截止日期+通知=智能提醒；标签+快捷输入=语法扩展
- **A-调整**：借鉴Apple Notes Quick Note（角落面板）、Flomo无压记录（极简模式）、Obsidian本地优先（存储架构）
- **M-修改**：放大→桌面悬浮窗始终可见；缩小→单输入框极简模式
- **P-另作他用**：状态流转复用为习惯打卡视图
- **E-消除**：消除登录门槛（免登录本地模式）；消除手动保存
- **R-重排**：先记录后分类（AI/规则自动建议）

### 5Why 根因追问（备忘录墓地问题）

**痛点**：用户创建了备忘后经常忘记看，导致备忘变成"备忘录墓地"
- **Why 1**：因为备忘没有主动提醒用户 → 方案：智能提醒引擎
- **Why 2**：因为只有设置了截止日期的才有提醒，普通备忘缺乏回顾机制 → 方案：每日智能回顾
- **Why 3**：因为用户记完就关闭应用，没有"再看一眼"的动机 → 方案：桌面悬浮窗Top3始终可见
- **Why 4（根因）**：产品只解决了"记录"问题，没有解决"回顾"和"执行"问题

### 反向脑暴关键逆转

| 恶化方案 | 逆转后的改进方案 |
|---------|-------------|
| 每次打开都弹登录框 | 免登录本地模式，渐进式引导注册 |
| 删除不可逆，误删就完蛋 | 回收站软删除，30天内可恢复 |
| 所有设备同时响通知 | 智能设备感知，只在活跃设备提醒 |
| 笔记越多越找不到 | 全文搜索+智能排序 |
